========================================
SkillMaster 推送到 GitHub 指南
========================================

你的 GitHub 仓库地址：
https://github.com/LeeSeaside/skillmaster.git

========================================
方法一：使用自动脚本（最简单）
========================================

Windows 用户：
1. 双击运行 "push-to-github.bat" 文件
2. 等待完成即可

macOS/Linux 用户：
1. 打开终端
2. 进入项目目录
3. 运行：chmod +x push-to-github.sh
4. 运行：./push-to-github.sh

========================================
方法二：手动执行（如果脚本失败）
========================================

打开命令行/终端，依次执行以下命令：

1. 初始化 Git（如果还没有）
   git init

2. 添加远程仓库
   git remote add origin https://github.com/LeeSeaside/skillmaster.git

3. 添加所有文件
   git add .

4. 提交更改
   git commit -m "Initial commit: SkillMaster v0.1.0"

5. 设置主分支
   git branch -M main

6. 推送到 GitHub
   git push -u origin main

========================================
注意事项
========================================

1. 首次推送可能需要输入 GitHub 用户名和密码
   - 用户名：你的 GitHub 用户名
   - 密码：使用 Personal Access Token（不是账号密码）

2. 如何获取 Personal Access Token：
   - 登录 GitHub
   - Settings → Developer settings → Personal access tokens
   - Generate new token (classic)
   - 勾选 "repo" 权限
   - 生成并复制 Token
   - 在推送时使用 Token 作为密码

3. 推荐使用 SSH 方式（更安全）：
   - 生成 SSH 密钥：ssh-keygen -t ed25519 -C "your_email@example.com"
   - 添加到 GitHub：Settings → SSH and GPG keys
   - 更改远程 URL：git remote set-url origin git@github.com:LeeSeaside/skillmaster.git

========================================
验证推送成功
========================================

推送成功后，访问以下地址查看：
https://github.com/LeeSeaside/skillmaster

你应该能看到所有文件已经上传。

========================================
后续更新
========================================

首次推送后，后续更新只需三步：

1. git add .
2. git commit -m "更新说明"
3. git push

========================================
常见问题
========================================

Q: 推送失败，提示认证错误？
A: 使用 Personal Access Token 作为密码，或配置 SSH

Q: 推送被拒绝？
A: 先拉取远程更改：git pull origin main --rebase

Q: 文件太大无法推送？
A: 检查 .gitignore 是否正确配置

Q: 如何查看推送状态？
A: git status

Q: 如何查看提交历史？
A: git log
